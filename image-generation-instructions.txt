================================================================================
                      2. IMAGE GENERATION (imagegen)
================================================================================

2.1 BASIC USAGE
---------------

Simple format - just provide a prompt:

    {"imagegen": "A serene mountain landscape at sunset"}

The system will use the default model (nano-banana) and generate one image.


2.2 FULL OPTIONS
----------------

Full format with all options:

    {"imagegen": {
        "prompt": "A futuristic cityscape with flying cars",
        "model": "nano-banana-pro",
        "n": 2,
        "size": "1024x1024",
        "aspectRatio": "16:9"
    }}

Parameters explained:
- prompt: (required) Description of the image to generate
- model: (optional) Which model to use (default: nano-banana)
- n: (optional) Number of images to generate, 1-4 (default: 1)
- size: (optional) Image dimensions (default: 1024x1024)
- aspectRatio: (optional) Aspect ratio like "16:9", "1:1", "4:3" (default: 1:1)


2.3 HASHTAG FORMAT
------------------

Alternative format using hashtags (user input only):

    #imagegen:A beautiful sunset over the ocean

With model specified:

    #imagegen:nano-banana-pro:A detailed portrait of a cat wearing a hat

================================================================================
                       3. IMAGE EDITING (imageedit)
================================================================================

Image editing allows you to modify existing images. Many models support
image-to-image generation:

MODELS THAT SUPPORT IMAGE EDITING:
- nano-banana (gemini-2.5-flash-image) - Fast, good quality
- nano-banana-pro (gemini-3-pro-image-preview) - Best quality
- flux-2-klein-4b - Fastest (up to 4 reference images)
- flux-2-klein-9b - Fast with better prompts (up to 4 refs)
- flux-2-pro - Production quality (up to 8 reference images)
- flux-2-max - Maximum quality (up to 8 reference images)


3.1 USING THE LAST IMAGE
------------------------

The simplest way to edit an image is to reference the last image in the
conversation. This works for:
- Images you just generated
- Images you uploaded

Simple format (automatically uses the last image and default model):

    {"imageedit": "Make the sky purple and add stars"}

**WITH SPECIFIC MODEL** - Very important! You can choose any model:

    {"imageedit": {
        "prompt": "Make the sky purple and add stars",
        "model": "flux-2-pro"
    }}

    {"imageedit": {
        "prompt": "Transform to oil painting style",
        "model": "flux-2-max"
    }}

    {"imageedit": {
        "prompt": "Add neon lights and cyberpunk effects",
        "model": "nano-banana-pro"
    }}

NOTE: If you provide a prompt but no explicit image reference, the system
will AUTOMATICALLY use the last image in the conversation. This is the most
convenient way to edit images.

With explicit flag:

    {"imageedit": {
        "prompt": "Add butterflies flying around",
        "model": "flux-2-pro",
        "use_last_image": true
    }}


3.2 REFERENCE IMAGES BY ID
--------------------------

Every image in the conversation has a unique ID (e.g., "img_001", "img_002").
You can reference specific images by their ID:

Single image by ID:

    {"imageedit": {
        "prompt": "Make this look like a watercolor painting",
        "image_id": "img_001"
    }}

Multiple images by IDs:

    {"imageedit": {
        "prompt": "Combine these two images into one scene",
        "image_ids": ["img_001", "img_003"]
    }}


3.3 PROVIDING BASE64 IMAGES
---------------------------

You can also provide images directly as base64 data URLs:

    {"imageedit": {
        "prompt": "Combine these images into one scene",
        "input_images": [
            "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA...",
            "data:image/jpeg;base64,/9j/4AAQSkZJRgABAQAA..."
        ]
    }}

Alternative key names for input_images:
- input_images
- inputImages
- reference_images

================================================================================
                       4. IMAGE IDENTIFIERS (IDs)
================================================================================

4.1 HOW IDs WORK
----------------

Every image in the conversation receives a unique identifier with a prefix
indicating the source:

- Generated images: gen_XXXXX (e.g., gen_00001, gen_00002)
- Uploaded images:  upl_XXXXX (e.g., upl_00001, upl_00002)
- Legacy format:    img_XXX (still supported for backwards compatibility)

Key features:
- IDs use 5 digits for large collections (up to 99,999 images)
- IDs are globally unique across the entire conversation
- Different prefixes make it clear where each image came from
- IDs persist when saving/loading conversations

Examples in order:
- First image generated:  gen_00001
- Second image uploaded:  upl_00002
- Third image generated:  gen_00003
- Fourth image uploaded:  upl_00004


4.2 VIEWING IDs IN THE UI
-------------------------

In the chat interface, images display:

- ID BADGE: Bottom-left corner shows the image ID (e.g., "gen_00001")
  - Click to copy the ID to clipboard
  - Useful for referencing in edit commands

- SOURCE BADGE: Top-right corner shows "Generated" or "Uploaded"
  - Purple = AI-generated image (gen_XXXXX)
  - Blue = User-uploaded image (upl_XXXXX)

- FULL VIEW: Click any image to open it in a new tab


4.3 REFERENCING IMAGES BY ID
----------------------------

Use image IDs to edit specific images:

    User: Edit gen_00001 to make it look vintage

    Agent response:
    {"imageedit": {
        "prompt": "Apply a vintage filter with sepia tones and film grain",
        "image_id": "gen_00001"
    }}

Combine multiple images:

    {"imageedit": {
        "prompt": "Create a collage from these images",
        "image_ids": ["gen_00001", "upl_00002", "gen_00003"]
    }}


4.4 IMAGE CONTEXT IN MESSAGES
-----------------------------

When you upload images, the system automatically adds context hints:

    [Attached image: upl_00001 (photo.jpg)]

This helps the LLM understand what images are present in your message.
The context includes:
- The unique image ID
- The original filename (for uploaded images)
- The source type (uploaded vs generated)

                       5. PARAMETERS REFERENCE
================================================================================

IMAGEGEN PARAMETERS:
--------------------
| Parameter    | Type     | Required | Default        | Description           |
|--------------|----------|----------|----------------|----------------------|
| prompt       | string   | YES      | -              | Image description    |
| model        | string   | no       | nano-banana    | Model to use         |
| n            | number   | no       | 1              | Number of images 1-4 |
| size         | string   | no       | 1024x1024      | Image dimensions     |
| aspectRatio  | string   | no       | 1:1            | Aspect ratio         |
| input_images | string[] | no       | []             | Reference images     |

IMAGEEDIT PARAMETERS:
---------------------
| Parameter      | Type     | Required | Default      | Description           |
|----------------|----------|----------|--------------|----------------------|
| prompt         | string   | YES*     | "Edit image" | Edit instructions    |
| model          | string   | no       | nano-banana  | Model to use         |
| input_images   | string[] | no       | []           | Base64 images        |
| use_last_image | boolean  | no       | auto**       | Use last image       |
| image_id       | string   | no       | -            | Reference by ID      |
| image_ids      | string[] | no       | []           | Reference multiple   |
| n              | number   | no       | 1            | Number of outputs    |

* prompt is the only truly required field
** use_last_image defaults to TRUE if no other image source is specified

PRIORITY ORDER FOR IMAGE SOURCE:
1. input_images (explicit base64 data)
2. image_id / image_ids (reference by ID)
3. use_last_image (automatic fallback)

SUPPORTED SIZES:
----------------
- 1024x1024 (square, default)
- 1792x1024 (landscape)
- 1024x1792 (portrait)
- 1536x1024 (landscape)
- 1024x1536 (portrait)

SUPPORTED ASPECT RATIOS:
------------------------
- 1:1 (square, default)
- 16:9 (widescreen)
- 9:16 (vertical/mobile)
- 4:3 (standard)
- 3:4 (vertical standard)
- 3:2 (photo)
- 2:3 (vertical photo)

================================================================================
                        8. TROUBLESHOOTING
================================================================================

ISSUE: "No image found to edit"
SOLUTION: You need to either:
- Generate an image first using {"imagegen": "..."}
- Upload an image before trying to edit
- Provide a valid image_id that exists in the conversation
- Provide explicit input_images in your imageedit command

ISSUE: "Image ID does not exist"
SOLUTION: Check the conversation for available image IDs:
- Click on images to see their ID badge
- Generated images use gen_XXXXX format
- Uploaded images use upl_XXXXX format
- IDs are globally unique and sequential
- If you cleared the chat, IDs reset

ISSUE: Image generation times out
SOLUTION: Image generation can take 30-60 seconds for complex prompts.
The system has a 120-second timeout. If it still times out:
- Try a simpler prompt
- Check your middleware connection
- Reduce the number of images (n parameter)

ISSUE: "[object Object]" appears instead of image
SOLUTION: This was a bug that has been fixed. If you see this:
- Refresh the page
- The fix ensures multimodal content (images) is preserved properly

ISSUE: Model returns 404 error
SOLUTION: The requested model may not be available in your setup:
- Check that the model is enabled in your Google Cloud project
- Try a different model (nano-banana is the most reliable)
- Check middleware logs for more details

ISSUE: Image quality is low
SOLUTION: Use a higher-quality model:
- nano-banana-pro for detailed work
- Use a more detailed prompt
- Avoid fast models for complex images

ISSUE: Image editing doesn't work
SOLUTION:
- Make sure you're using a model that supports editing
- Ensure there's an image to edit (check the ID or upload one)
- Try being more specific in your edit prompt
- Check that the image_id you referenced exists

ISSUE: Multiple images (n:2) not working with edit
SOLUTION: This has been fixed. The system now:
- Auto-detects when no image source is provided
- Automatically uses the last image if only prompt is given
- Supports the n parameter for multiple output images

ISSUE: Image description not working / no response after upload
SOLUTION: When you upload an image and ask for description:
- Check that VLM (Vision Language Model) is configured in settings
- The system now adds context hints like "[Attached image: upl_00001 (photo.jpg)]"
- Make sure the VLM endpoint is working (check middleware logs)
- Try a simple prompt like "Describe this image" or "What do you see?"
- Check browser console for any errors

ISSUE: Uploaded and generated images have same ID
SOLUTION: This has been fixed. Now:
- Generated images use gen_XXXXX prefix
- Uploaded images use upl_XXXXX prefix
- IDs are globally unique across all images in the conversation
- The system scans full message history to avoid collisions

                           NOTES FOR DEVELOPERS

The image generation system works through these files:
- routes/api/imagegen.ts - API endpoint (120s timeout)
- islands/ChatIsland/services/apiService.ts - Frontend API calls
- islands/ChatIsland/utils/triggerParsing.ts - JSON trigger parsing
- islands/ChatIsland/services/streamService.ts - Trigger execution & ID generation
- islands/ChatIsland/types.ts - Type definitions (AutoTrigger, ImageContentPart)
- components/ChatTemplate.tsx - Image display with ID badges
- components/ImageUploadButton.tsx - Upload with progress tracking & ID assignment
- components/PdfUploadButton.tsx - PDF upload with progress tracking
- Admin_Bud-E/main.py - Middleware endpoint
- Admin_Bud-E/providers.py - Provider implementations

IMAGE ID FORMAT (NEW):
----------------------
- Generated images: gen_XXXXX (e.g., gen_00001)
- Uploaded images:  upl_XXXXX (e.g., upl_00001)
- Legacy format:    img_XXX (still supported for backwards compatibility)

The ID format uses:
- 3-letter prefix to indicate source (gen/upl/img)
- 5-digit zero-padded number for large collections
- Global uniqueness across the entire conversation

Images are stored as base64 data URLs with metadata in the message content array:
{
  "role": "assistant",
  "content": [
    {"type": "text", "text": "Generated image..."},
    {
      "type": "image_url",
      "image_url": {"url": "data:image/png;base64,..."},
      "id": "gen_00001",
      "source": "generated",
      "timestamp": 1706900000000
    }
  ]
}

For uploaded images, additional metadata is included:
{
  "type": "image_url",
  "image_url": {"url": "data:image/png;base64,...", "detail": "high"},
  "id": "upl_00001",
  "source": "uploaded",
  "timestamp": 1706900000000,
  "filename": "photo.jpg"
}

Image ID Functions (in streamService.ts):
- generateImageId(messages, index) - Creates next gen_XXXXX ID
- findHighestImageIdForPrefix(messages, prefix) - Finds max ID for prefix
- findImageByIdInMessages(messages, id) - Looks up image by any ID format
- getAllImageIds(messages) - Lists all image IDs with metadata
- buildImageContextHints(images) - Creates text hints for LLM

Image Upload Functions (in ImageUploadButton.tsx):
- findAllHighestImageIds(messages) - Scans all formats (gen/upl/img)
- generateUploadImageId(existingMax, index) - Creates upl_XXXXX IDs

This format is preserved when:
- Saving to localStorage
- Downloading chat as JSON
- Uploading/restoring chats

2. gemini-2.5-flash-image (priority 2) - Fast fallback
3. dall-e-3 (priority 10) - Emergency fallback

IMAGEGEN PARAMETERS:
- prompt          : (required) Text description of the image
- model           : Model name (see table above)
- n               : Number of images (1-4)
- size            : "1024x1024", "1792x1024", "1024x1792"
- aspectRatio     : "1:1", "16:9", "9:16", "4:3", "3:2"

IMAGEEDIT PARAMETERS:
- prompt          : (required) What changes to make
- model           : Model name (see table above)
- use_last_image  : true/false - edit the most recent image
- image_id        : "gen_00001" - edit specific image
- image_ids       : ["id1", "id2"] - combine multiple images
- input_images    : ["data:image/..."] - provide base64 images
- n               : Number of output images (1-4)
